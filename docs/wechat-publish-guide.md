## 准备阶段

### 必备材料

- ✅ 未注册过微信的邮箱（用于注册小程序）
- ✅ 身份证（个人开发者需要实名认证）
- ✅ 已绑定的微信账号
- ✅ Cocos Creator 3.8.8 或更高版本
- ✅ 项目已完成开发

### 开发环境

- macOS 10.14+ 或 Windows 7+
- Node.js v16+（用于 Cocos Creator）
- Cocos Creator 3.8.8
- 微信开发者工具（最新稳定版）

## 注册微信小游戏账号

### 步骤 1：访问微信公众平台

1. 打开浏览器，访问：https://mp.weixin.qq.com/
2. 点击页面右上角的 **"立即注册"**

### 步骤 2：选择账号类型

1. 在注册页面，选择 **"小程序"**
2. 在子分类中选择 **"小游戏"**
3. 点击 **"前往注册"**

### 步骤 3：填写注册信息

#### 基础信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 邮箱 | 未注册过微信的邮箱 | your-email@example.com |
| 密码 | 8-20位，包含字母和数字 | Abcd1234 |
| 确认密码 | 与密码一致 | Abcd1234 |

#### 验证邮箱

1. 登录邮箱，查看微信发送的验证邮件
2. 点击邮件中的 **"激活账号"** 链接
3. 页面提示激活成功

### 步骤 4：扫码绑定微信

1. 使用微信扫描页面上的二维码
2. 在手机上确认绑定
3. 绑定成功后，账号将与你的微信关联

### 步骤 5：实名认证

个人开发者必须完成实名认证：

1. 点击 **"前往认证"**
2. 选择 **"个人"** 认证类型
3. 上传身份证照片：
   - 正面照片（清晰可见）
   - 反面照片（清晰可见）
4. 进行人脸识别验证：
   - 按照提示完成人脸动作
   - 确保 光线充足、面部清晰
5. 等待审核（通常 1-5 分钟）

### 步骤 6：获取 AppID

认证通过后，获取 AppID：

1. 登录微信公众平台
2. 点击左侧菜单 **"设置"**
3. 选择 **"基本设置"**
4. 在页面中找到 **"AppID (小程序ID)"**
5. **复制并保存 AppID**（格式：wxXXXXXXXXXXXXXXXX）

> ⚠️ 重要：AppID 是游戏的唯一标识，请妥善保存，后续构建需要用到。

## 构建微信小游戏版本

### 步骤 1：打开 Cocos Creator 项目

1. 打开 **Cocos Creator 3.8.8**
2. 点击 **"打开其他项目"**
3. 选择项目目录：`/Users/xuefanfei/Ffcs/aiproject/game-t/`
4. 等待项目加载完成

### 步骤 2：配置构建参数

1. 在 Cocos Creator 顶部菜单，点击 **"项目"**
2. 选择 **"构建发布"**
3. 在右侧面板中配置以下参数：

#### 基本配置

| 参数项 | 值 | 说明 |
|--------|-----|------|
| 平台 | 微信小游戏 | 目标平台 |
| 场景 | 勾选 Launch、Game | 包含的场景 |
| 游戏名称 | 往死里挤 | 显示名称 |
| 游戏包名 | the-squeeze | 包标识符 |
| AppID | 填写你的 AppID | 从微信公众平台获取 |
| 初始场景 | Launch | 启动场景 |

#### 高级配置

点击 **"高级配置"** 按钮，设置以下参数：

| 参数项 | 值 | 说明 |
|--------|-----|------|
| 渲染后端 | WEBGL | 使用 WebGL 渲染 |
| 启用内联 SpriteFrame | 是 | 优化性能 |
| 启用压缩纹理 | 否 | 首次发布可不启用 |
| 生成 JSB | 否 | 原生相关，小游戏不需要 |
| 自动合并子包 | 是 | 减少资源文件数 |

### 步骤 3：构建项目

1. 配置完成后，点击 **"构建"** 按钮
2. 等待构建过程（通常 1-3 分钟）
3. 构建进度条完成后，会提示 **"构建成功"**

### 步骤 4：查看构建结果

1. 构建完成后，点击 **"打开构建文件夹"** 按钮
2. 构建结果目录：`/Users/xuefanfei/Ffcs/aiproject/game-t/build/wechatgame/`
3. 目录结构：
   ```
   wechatgame/

   ├── game.js
   ├── js/
   │   ├── cocos-js
   │   └── ...
   ├── assets/
   ```

## 真机测试

### 为什么需要真机测试？

- 模拟器无法完全模拟真实设备
- 真机性能与模拟器有差异
- 触摸操作与鼠标不同
- 需要测试在不同设备上的表现

### 步骤 1：生成预览二维码

1. 在微信开发者工具中，点击工具栏的 **"预览"** 按钮
2. 等待代码编译和上传（约 10-30 秒）
3. 编译成功后，会显示二维码

### 步骤 2：使用手机扫码

1. 打开微信（手机）
2. 扫描开发者工具中的二维码
3. 点击 **"进入小程序"**
4. 游戏开始运行

### 步骤 3：完整游戏流程测试

按照 [本地测试](#本地测试) 中的清单，在真机上完成所有测试项。

#### 真机特别关注的点：

| 关注点 | 说明 |
|--------|------|
| 触摸响应 | 触摸是否灵敏，是否有延迟 |
| 画面流畅度 | 真实 FPS 是否达到 60 |
| 内存占用 | 真实内存使用情况 |
| 电池消耗 | 玩一段时间后的电池消耗 |
| 发热情况 | 长时间玩是否会过热 |
| 震动效果 | 真实震动反馈是否正常 |
| 分享功能 | 真实分享是否成功 |

- 小米

在真机上，微信开发者工具会实时显示性能数据：

### 常见真机问题及解决方案

#### 问题 1：触摸不灵敏

**原因**：
- 点击检测区域过小
- 触摸事件处理延迟

**解决方案**：
- 增加 Ball 的碰撞体半径
- 优化 InputManager 的触摸处理逻辑

#### 问题 2：FPS 下降严重

**原因**：
- 物理计算过重
- 渲染对象过多

**解决方案**：
- 减少 Ball 数量
- 降低物理迭代次数
- 启用对象池

#### 问题 3：内存占用过高

**原因**：
- 纹理资源过大
- 对象频繁创建销毁

**解决方案**：
- 使用更小的纹理尺寸
- 启用对象池复用
- 及时释放不再使用的资源

#### 问题 4：震动不工作

**原因**：
- 微信震动 API 兼容性问题
- 权限未开启

**解决方案**：
- 检查 `VibrationManager.ts` 中的平台判断
- 在真机设置中开启震动权限

2. 录制游戏过程

- 添加文字说明（可选）

### 2. 游戏图标

#### 要求

- 尺寸：**512 x 512** 像素
- 格式：PNG
- 设计：圆形或圆角方形
- 背景：透明背景

**主题**：
- 使用游戏元素（团子 + 容器）

- 使用游戏中的马卡龙色系
- 对比度适中，易于识别
- 避免过多颜色

**布局**：
- 主体居中
- 留有适当留白
- 避免文字过多

#### 制作工具

- **专业工具**：Photoshop、Sketch、Figma
- **在线工具**：Canva、稿定设计
- **AI 生成**：使用 AI 图像生成工具

## 提交审核

### 步骤 1：上传代码

#### 1.1 在微信开发者工具中上传

1. 在微信开发者工具中，点击工具栏的 **"上传"** 按钮
2. 填写版本信息：

| 字段 | 值 | 说明 |
|------|-----|------|
| 版本号 | 1.0.0 | 主版本号.次版本号.修订号 |
| 项目备注 | 初次发布 | 本次更新的说明 |

3. 点击 **"上传"** 按钮
4. 等待上传完成（约 1-2 分钟）

#### 1.2 上传成功提示

上传成功后，会显示：

```
✓ 代码上传成功
版本号：1.0.0
上传时间：2026-02-18 14:30:00
```

### 步骤 2：登录微信公众平台

1. 打开浏览器，访问：https://mp.weixin.qq.com/
2. 使用注册小游戏时的微信扫码登录
3. 登录成功后进入管理后台

### 步骤 3：进入版本管理

1. 在左侧菜单中，点击 **"开发"**
2. 选择 **"版本管理"**
3. 在页面中看到刚上传的版本（1.0.0）

### 步骤 4：完善游戏信息

#### 4.1 基本信息

点击 **"完善游戏信息"** 按钮，填写：

| 字段 | 值 | 说明 |
|------|-----|------|
| 游戏名称 | 往死里挤 | 与构建时的名称一致 |
| 游戏简介 | 一款休闲物理益智小游戏... | 参考准备的材料 |
| 游戏类目 | 休闲益智 | 选择最匹配的类目 |
| 游戏标签 | 物理、益智、闯关、休闲 | 最多 10 个标签 |

#### 4.2 上传图标和截图

**游戏图标**：
- 点击 **"上传图标"** 按钮
- 选择准备好的 512x512 图标
- 预览确认后点击确定

**游戏截图**：
- 点击 **"添加截图"** 按钮
- 依次上传 5 张截图
- 确保顺序合理（从入口到玩法到结算）

#### 4.3 游戏介绍（可选）

可以添加更详细的介绍，支持 Markdown 格式：

```markdown
# 《往死里挤》

一款简单上手的物理益智游戏！

## 玩法说明
1. 拖动团子到容器中
2. 将所有团子挤进容器
3. 达到 80% 填充率即可过关

## 游戏特色
- 20 个精心设计的关卡
- 真实的物理模拟
- 特殊关卡：移动容器、长条团子
- 多样化的评价系统

## 玩家评价
"太好玩了！" - 玩家A
"简单又上瘾！" - 玩家B
```

### 步骤 5：测试账号（可选）

如果游戏需要登录，填写测试账号信息：

| 字段 | 值 |
|------|-----|
| 测试账号 | test@example.com |
| 测试密码 | Test1234 |

个人开发者通常不需要填写此部分。

### 步骤 6：补充说明（可选）

填写一些补充说明，帮助审核人员了解游戏：

```
本游戏是一款纯休闲益智小游戏，不涉及任何付费功能、
广告或敏感内容。游戏内容健康积极，适合全年龄层用户。

如有任何问题，请随时联系开发者。
```

### 步骤 7：提交审核

1. 确认所有信息填写正确
2. 点击 **"提交审核"** 按钮
3. 阅读并同意《微信小程序平台运营规范》
4. 点击 **"确认提交"**

### 步骤 8：等待审核

#### 审核时间

- **个人开发者**：通常 1-3 个工作日
- **企业开发者**：通常 3-7 个工作日

#### 审核状态

在 **"版本管理"** 页面可以看到审核状态：

| 状态 | 说明 | 操作 |
|------|------|------|
| 审核中 | 正在审核，请耐心等待 | 无 |
| 审核通过 | 可以发布上线 | 点击"发布" |
| 审核驳回 | 需要修改后重新提交 | 查看驳回原因 |

#### 审核期间可以做什么？

审核期间，你可以：

- ✅ 继续开发新功能
- ✅ 使用"体验版"功能分享给朋友测试
- ✅ 查看审核进度
- ✅ 准备下一版本的更新内容

### 步骤 9：审核结果处理

#### 情况 1：审核通过

恭喜！你的游戏可以上线了。

**发布上线步骤**：

1. 在审核通过的状态下，点击 **"发布"** 按钮
2. 填写发布说明（可选）：
   ```
   《往死里挤》v1.0.0 正式上线！
   包含 20 个精心设计的关卡，快来挑战吧！
   ```
3. 点击 **"确认发布"** 按钮
4. 发布成功后，游戏正式上线

**用户如何找到你的游戏**：

1. 在微信中搜索"往死里挤"
2. 通过朋友分享的链接
3. 在小游戏分类中浏览

#### 情况 2：审核被驳回

不要气馁！查看驳回原因并修改。

**查看驳回原因**：

1. 在 **"版本管理"** 页面
2. 点击被驳回的版本
3. 查看 **"驳回原因"** 详细说明

**常见驳回原因及解决方案**：

| 驳回原因 | 解决方案 |
|----------|----------|
| 游戏内容不完整 | 添加更多关卡或完善功能 |
| 截图不符合要求 | 重新制作符合尺寸的截图 |
| 图标不符合规范 | 重新设计符合要求的图标 |
| 界面布局问题 | 优化 UI 布局和视觉设计 |
| 功能不正常 | 修复 Bug 后重新提交 |
| 内容违规 | 修改违规内容，确保健康合规 |

**重新提交**：

1. 修改代码或内容
2. 在 Cocos Creator 中重新构建
3. 在微信开发者工具中重新上传
4. 在微信公众平台重新提交审核

## 常见问题

### Q1: 微信开发者工具无法打开项目？

**可能原因**：
- AppID 填写错误
- 项目目录选择错误
- 构建输出不完整

**解决方案**：
1. 确认 AppID 正确（从微信公众平台复制）
2. 确认选择的是 `build/wechatgame/` 目录
3. 在 Cocos Creator 中重新构建

### Q2: 真机预览时白屏？

**可能原因**：
- 代码错误
- 资源加载失败
- 权限问题

**解决方案**：
1. 查看微信开发者工具的 Console，检查错误信息
2. 查看 Network，检查资源加载状态
3. 确保手机和电脑在同一网络环境

### Q3: 审核被驳回了怎么办？

**解决方案**：
1. 仔细阅读驳回原因
2. 按照要求修改问题
3. 重新构建和上传
4. 重新提交审核
5. 保持耐心，多次尝试很正常

### Q4: 游戏包大小超过 4MB 怎么办？

**解决方案**：
1. 启用纹理压缩（ASTC/ETC2）
2. 移除未使用的资源
3. 开启分包加载
4. 优化代码体积（启用压缩和混淆）

### Q5: 如何分享游戏给朋友？

**方法 1：使用体验版**
1. 在微信开发者工具中生成预览二维码
2. 朋友扫码即可体验（7 天内有效）

**方法 2：使用分享功能**
1. 游戏上线后
2. 在游戏中使用分享功能
3. 朋友点击分享链接即可访问

### Q6: 震动功能在真机上不工作？

**可能原因**：
- 微信震动 API 版本兼容性问题
- 手机不支持震动
- 震动权限未开启

**解决方案**：
1. 更新微信开发者工具到最新版本
2. 检查 `VibrationManager.ts` 的实现
3. 在手机设置中开启震动权限
4. 检查手机是否支持震动

### Q7: 如何获取用户数据？

**注意**：
- 微信小游戏对用户数据有限制
- 需要用户授权才能获取

**解决方案**：
1. 使用微信 `wx.getUserInfo` 获取基础信息
2. 使用 `wx.getStorage` 保存用户进度
3. 遵守微信隐私政策

### Q8: 如何接入广告变现？

**步骤**：
1. 在微信公众平台开通广告组件
2. 查阅微信广告文档：https://developers.weixin.qq.com/minigame/dev/guide/open-ability/ad/ad-overview.html
3. 在代码中集成广告 SDK
4. 测试广告显示

### Q9: 如何接入支付功能？

**注意**：
- 个人开发者**不支持**支付功能
- 需要企业资质

**解决方案**：
- 个人开发者可以使用激励视频广告
- 企业开发者可以接入虚拟支付

### Q10: 审核需要多长时间？

**时间**：
- 个人开发者：1-3 个工作日
- 企业开发者：3-7 个工作日

**加速技巧**：
- 确保游戏功能完整
- 提供详细的补充说明
- 提前阅读审核规范
